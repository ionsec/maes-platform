.. _api-alerts:

Alerts
======

Manage security alerts generated by the analyzer or created manually.

List Alerts
-----------

.. http:get:: /api/alerts

   :query page: Page number (default 1)
   :query limit: Items per page (default 20, max 1000)
   :query status: ``new``, ``acknowledged``, ``investigating``, ``resolved``, ``false_positive``
   :query severity: ``low``, ``medium``, ``high``, ``critical``
   :query category: Filter by alert category

Get Alert
---------

.. http:get:: /api/alerts/(id)

Acknowledge Alert
-----------------

.. http:put:: /api/alerts/(id)/acknowledge

   Move an alert from ``new`` to ``acknowledged``. Requires ``canManageAlerts`` permission.

Assign Alert
------------

.. http:put:: /api/alerts/(id)/assign

   Assign an alert to a user. Requires ``canManageAlerts`` permission.

   **Request Body:**

   .. code-block:: json

      { "assignedTo": "user-uuid" }

Resolve Alert
-------------

.. http:put:: /api/alerts/(id)/resolve

   Resolve or mark an alert as false positive. Requires ``canManageAlerts`` permission.

   **Request Body:**

   .. code-block:: json

      {
        "status": "resolved",
        "resolutionNotes": "Confirmed authorized activity"
      }

   ``status`` must be ``resolved`` or ``false_positive``.

Create Alert
------------

.. http:post:: /api/alerts

   Create a new alert (typically called by the analyzer service).

   **Request Body:**

   .. code-block:: json

      {
        "title": "Suspicious Login Activity",
        "description": "Multiple failed logins from unusual location",
        "severity": "high",
        "source": "ual_analysis",
        "organizationId": "uuid",
        "data": {
          "finding": {
            "type": "brute_force",
            "category": "authentication",
            "affectedEntities": { "users": ["user@domain.com"] },
            "mitreAttack": { "tactics": ["Credential Access"], "techniques": ["T1110"] }
          }
        }
      }

Alert Statistics
----------------

.. http:get:: /api/alerts/stats/summary

   Get aggregated alert statistics by severity and status.

Delete Alert
------------

.. http:delete:: /api/alerts/(id)

   Delete an alert. Requires ``canManageAlerts`` permission. Deletion is audited.
