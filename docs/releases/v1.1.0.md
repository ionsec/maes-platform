# MAES Platform v1.1.0

## Summary

`v1.1.0` combines upstream extractor/analyzer alignment with a full security remediation pass across application dependencies, runtime defaults, and container configuration.

## Security Fixes

- Cleared known production dependency vulnerabilities across `api`, `frontend`, `services/analyzer`, `services/extractor`, and `services/compliance`.
- Removed hardcoded fallback values for `SERVICE_AUTH_TOKEN`, `ENCRYPTION_KEY`, and certificate passwords.
- Removed the seeded `admin@maes.local / admin123` bootstrap account from `database/init.sql`.
- Disabled Docker log scraping by default unless operators explicitly opt in.
- Tightened CSP and production CORS defaults.

## Dependency Upgrades

- `axios` upgraded to `1.15.0` across services.
- `express`, `express-validator`, `multer`, `socket.io`, `react-router-dom`, `sequelize`, `puppeteer`, and related vulnerable transitives updated or overridden to patched versions.
- Lockfiles regenerated after remediation to ensure clean production audit results.

## Docker And Deployment Hardening

- Replaced floating Node, nginx, Redis, and TimescaleDB image references with pinned digests or fixed tags.
- Removed `docker-cli` from the API image.
- Added missing `.dockerignore` files and switched service installs to production-only dependency mode.
- Added health checks and `dumb-init` where appropriate.
- Compose now requires explicit secret values for core services.

## Platform Updates

- Added a shared capability registry for extractor/API/frontend parity.
- Added support for newer upstream queueable extraction commands.
- Corrected extractor command mappings for OAuth permissions, risky users, and full evidence collection.

## Breaking / Operational Changes

- `.env` values are now mandatory for:
  - `POSTGRES_PASSWORD`
  - `REDIS_PASSWORD`
  - `JWT_SECRET`
  - `SERVICE_AUTH_TOKEN`
  - `ENCRYPTION_KEY`
  - `CERT_PASSWORD`
  - `GRAFANA_PASSWORD`
- There is no default administrator account after bootstrap.
- `/api/system/logs` returns `503` unless Docker log access is explicitly enabled.

## Upgrade Steps

1. Pull the new release.
2. Update `.env` with all required secrets.
3. If upgrading an existing deployment, ensure the old seeded admin account is replaced with a real administrator and rotate any known placeholder credentials.
4. Rebuild and restart the stack:

```bash
docker compose up -d --build
```

5. Verify:

```bash
docker compose ps
docker compose logs api --tail=100
docker compose logs extractor --tail=100
```

## Verification Performed

- `npm audit --omit=dev --json` returned zero production vulnerabilities for all Node projects.
- Frontend production build completed successfully.
- Modified backend/service files passed syntax checks.

Image-level scanning was not executed with authenticated Docker Scout in this environment; base-image pinning and in-repo hardening were still applied.
